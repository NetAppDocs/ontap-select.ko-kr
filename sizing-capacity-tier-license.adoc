= ONTAP Select Capacity Tiers 라이센스 크기 조정 시 고려 사항
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
Capacity Tiers 라이센스는 각 ONTAP Select 노드에서 관리하는 기본 디스크 용량을 기준으로 하며, 노드, HA 쌍 또는 클러스터의 사용 가능 용량을 기준으로 하지 않습니다. 각 Capacity Tiers 라이센스는 지정된 용량에 대해 개별 노드 일련 번호에 할당되며, 관련 지원 계약이 필요합니다.

Capacity Tiers 라이센스의 라이센스 용량은 최소 1TiB부터 시작하여 전체 TiB 단위로 제공됩니다. 라이센스 용량은 사용 가능한 공간에 오버헤드 계수를 곱하여 계산합니다. 오버헤드 계수는 단일 노드 클러스터와 다중 노드(HA 쌍 기반) 클러스터 간에 차이가 있습니다.

* 단일 노드 클러스터의 오버헤드 계수는 1.13입니다.
* 다중 노드 클러스터의 오버헤드 계수는 2.67입니다


구축에 필요한 용량이 충분한지 확인하려면 사용 가능한 공간을 기준으로 라이센스 용량을 계산하십시오.

[cols="25,30,45"]
|===
| 설명 | 단일 노드 | HA 쌍 


| 사용 가능한 공간 | 10TiB | 노드당 5TiB HA 쌍당 10TiB 


| 라이센스 공간 | 12 TiB(10 x 1.13 = 11.3, 다음 전체 TiB로 반올림) | 14TiB 라이센스 2개(5 x 2.67 = 13.35, 다음 전체 TiB로 반올림) 


| 노드당 최대 사용 가능 공간 *참고*: 노드 또는 HA 쌍당 최대 관리 가능 공간은 400 TiB입니다. | 353 TiB (400 / 1.13)  a| 
HA 쌍의 노드당:

* 74TiB[(400/2)/2.67]


다중 노드 클러스터의 HA 쌍당:

* 2노드: 148TiB(2 x 74)
* 4노드: 296TiB(4 x 74)
* 6노드: 444TiB(6 x 74)
* 8노드: 592TiB(8 x 74)
