= ONTAP Select Capacity Pools 라이센스 크기 조정 시 고려 사항
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
Capacity Pool 라이센스의 라이센스 용량은 최소 2TB의 전체 TB 단위입니다. 라이센스 용량은 사용 가능한 공간에 오버헤드 계수를 곱하여 계산합니다. 오버헤드 계수는 단일 노드 클러스터와 다중 노드(HA 쌍 기반) 클러스터 간에 차이가 있습니다.

* 단일 노드 클러스터의 오버헤드 계수는 1.13입니다.
* 다중 노드 클러스터의 오버헤드 계수는 2.67입니다


구축에 필요한 용량이 충분한지 확인하려면 사용 가능한 공간을 기준으로 라이센스 용량을 계산해야 합니다.

[cols="25,30,45"]
|===
| 설명 | 단일 노드 | HA 쌍 또는 HA 쌍과 단일 노드의 혼합 


| 사용 가능한 공간 | 20 TB | 단일 노드 클러스터당 10TB, HA 쌍의 경우 20TB 


| 라이센스 공간 | 23TB(20 x 1.13 = 22.6, 다음 전체 TB로 반올림) | 54TB(20 x 2.67 = 53.4, 다음 전체 TB로 반올림) 


| 노드당 최대 사용 가능 공간 *참고*: 노드 또는 HA 쌍당 최대 관리 공간은 400TB입니다. | 353TB(400 / 1.13)  a| 
클러스터 내 단일 노드당:

149TB(400 / 2.67)

클러스터의 HA 쌍당:

298TB(2 x 149)
